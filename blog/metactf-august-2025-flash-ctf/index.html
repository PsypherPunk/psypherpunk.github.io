On The Grid

I was hanging out on Granite Beach when I found a message in a bottle, it didn’t say very much though. Can you help me figure out where it came from? Here’s the transcript:

Help! 4V FH 246 677

The answer to the challenge is the name of the location the message was most likely written at.

Some searching suggests that “FV…” might be a “Military Grid Reference System”. Given the name of the challenge, that sounds promising.

This site, MGRS Converter: Military Grid Reference System, converts that value to a latitude of 56.53711577 and longitude of -156.97382099, which Google Maps shows to be Foggy Cape.

Super Quick Logic Invitational

I’ll take “Rather Vexing” for 500 Alex.

This new trivia game is pretty fun, but one of the challenges is impossible! “What is the flag for this CTF challenge?”, how would I know?! Maybe you’ll fare better?

The site presents “201 unique problems to solve each game, you’ll never believe what’s the answer to challenge 201!”, presumably the flag…?

“What is the largest prime factor of 625?”. After a surprising number of lucky guesses, it seems that the classic ' OR 1=1 -- works—the site is vulnerable to SQLi.

Using a classic UNION attack, via 'UNION SELECT 'A', 'B', 'C', 'D' --, shows that there are 4 parameters being returned. However, those values don’t show up until the site’s 30 second countdown has completed, at which point C shows up in the Answer value.

Thankfully, forcing an error by closing the ' and passing in any ol’ garbage, like '!"£$%^&*()_, results in an error:

{
    "error":"unrecognized token: \"!\"",
    "query":"SELECT * FROM problems WHERE id = 164 AND answer = ''!\"\u00a3$%^&*()_'"
}

…revealing both the table and column of interest. Putting those discoveries together and passing:

' UNION SELECT '', '', (SELECT answer FROM problems WHERE id = 201), '' --

…results in a page containing:

Answer: MetaCTF{wh4t_1s_7h3_fl4g_f0r_7hi5_ch5l1eng3}

Spreadsheet

I can’t afford Office 365 so I’ve decided to roll my own. What could possibly go wrong!

My spreadsheet application is a work in progress but feel free to give it a try. You can download my program here.

The “spreadsheet” app. is a TUI tool allowing editing, loading, saving and printing of—as you might guess—a spreadsheet:

❯ ./spreadsheet.bin b
Options: (P)rint, (E)dit, (L)oad, (S)ave, (Q)uit

As can be seen from the print_spreadsheet function (courtesy of Ghidra), it actually occupies a static 10x10 array (of char *):

void print_spreadsheet(void)

{
  int row;
  int col;

  putchar(9);
  for (col = 0; col < 10; col = col + 1) {
    printf("%c\t",(ulong)(col + 0x41));
  }
  putchar(10);
  for (row = 0; row < 10; row = row + 1) {
    printf("%2d\t",(ulong)(row + 1));
    for (col = 0; col < 10; col = col + 1) {
      printf("%4s\t",spreadsheet[row][col]);
    }
    putchar(10);
  }
  return;
}

The filename, spreadsheet.csv, is also static and, more importantly, its pointer sits right next to the above array on the stack.
